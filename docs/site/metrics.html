Coherence Metrics

Four composable metric points for observability.

coherence_score 90/100 drift_density 0.0 authority_coverage 1.0 memory_coverage 1.0

Metric Definitions

MetricUnitDescription
coherence_score score (0-100) Weighted composite of policy adherence, outcome health, drift control, and memory completeness
drift_density ratio Number of drift signals divided by episode count. Lower is better. Zero means no drift detected.
authority_coverage ratio Fraction of claims in the memory graph that have a valid authority grant. 1.0 means full coverage.
memory_coverage ratio Fraction of expected episodes that appear in the memory graph. 1.0 means all decisions are remembered.

CLI Usage

$ coherence metrics src/core/examples/ --json
{
  "metrics": [
    {"name": "coherence_score",    "value": 90.0,  "unit": "score"},
    {"name": "drift_density",      "value": 0.0,   "unit": "ratio"},
    {"name": "authority_coverage",  "value": 1.0,   "unit": "ratio"},
    {"name": "memory_coverage",    "value": 1.0,   "unit": "ratio"}
  ]
}
$ coherence agent metrics --agent fraud-detector-v3
