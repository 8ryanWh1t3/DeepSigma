Design Principles — Hard Law

No self-certification of closure No closure without owner + patch + evidence No evidence editing after review lock No overwrite of historical canon — append, version, reopen only Authority must be role-bound and state-bound

Platform Constraints

Delegation: non-delegable queries limited to 500-2000 rows Unique permissions: recommended limit 5,000 per list Power BI refresh: shared capacity 2-hour limit Use indexed columns for delegable filters
Phase 3 decision: if lists get large or relationships get complex, move core objects to Dataverse.

Power Automate 7 Flows

Flow 1: Owner Assignment Notification

Trigger→ Finding Status changes to "Assigned"
Action→ Send email to OwnerPerson with finding details + due date
Notify→ Teams message to PM channel

Flow 2: Review Request

Trigger→ Finding Status changes to "In Review"
Action→ Send email to ReviewerPerson with finding summary + BRQ
Action→ Create Review record in Reviews list
Notify→ Teams message to Board channel

Flow 3: Closure Request Approval

Trigger→ ClosureRequested = Yes
Action→ Start approval with ReviewerPerson
Action→ If approved: set Status = "Closed", ClosureApproved = Yes
Action→ If rejected: set Status = "In Review", add review note
Notify→ Email owner with decision

Flow 4: Reopen Alert

Trigger→ Reopened = Yes
Action→ Set Status = "Reopened"
Notify→ Email owner + reviewer + executive channel

Flow 5: Weekly Board Digest

Trigger→ Recurrence: Monday 0800
Action→ Query: findings where Status = "In Review" and BRQ >= 0.6
Notify→ Email digest to Board members with finding count, top severity, overdue items

Flow 6: KPI Refresh / Rollup

Trigger→ Recurrence: every 4 hours
Action→ Compute BRQ, CG, DES, DEG, EDI, PPI for each open finding
Action→ Write computed values back to Findings list
Keep the frontend clean: backend computes truth, frontend displays it. Do not compute KPIs inside Power Apps formulas.

Flow 7: Evidence Lock on Review Start

Trigger→ Review record created (ReviewStatus = "Active")
Action→ Set Locked = Yes for all Evidence records linked to this finding
Enforces: no evidence editing after review lock. Hard governance rule.

DAX Measures — Lead-6 KPIs

PAR — Patch Adhesion Rate
PAR = DIVIDE( COUNTROWS(FILTER(Findings, Findings[Status] = "Closed" && Findings[Reopened] = FALSE())), COUNTROWS(FILTER(Findings, Findings[Status] = "Closed")), 0 )
Durable closures / total closures. Higher = patches holding.
FCR — False Closure Rate
FCR = DIVIDE( COUNTROWS(FILTER(Findings, Findings[Reopened] = TRUE())), COUNTROWS(FILTER(Findings, Findings[ClosureApproved] = TRUE())), 0 )
Reopened / total closures. Lower = closures are real.
BRQ Average — Board Readiness Quotient
BRQ_Avg = AVERAGE( FILTER(Findings, Findings[Status] = "In Review"), Findings[BRQ] )
Average readiness score across in-review items.
CIS — Closure Integrity Score
CIS = 0.25 * [PAR] + 0.20 * [EDI_Norm] + 0.25 * (1 - [FCR]) + 0.15 * (1 - [DEG_Norm]) + 0.15 * [PPI_Avg]
Flagship composite. CIS > 0.75 = governance loop is real.
Open Findings
OpenFindings = COUNTROWS( FILTER(Findings, Findings[Status] <> "Closed") )
Avg DEG — Decision-to-Evidence Gap
AvgDEG = AVERAGEX( FILTER(Findings, NOT(ISBLANK(Findings[DEGDays]))), Findings[DEGDays] )
Average days between decision and evidence. Lower = faster proof.

Embedded Power Apps Visual

Use the Power Apps visual in Power BI to embed the Canvas app. This lets a leader see a red finding in Power BI, click into the embedded app, and take action without leaving the report context. The Power Apps visual passes context-aware data to the app.
Build Sequence — 8 Phases
Order matters. Build the record workflow before the dashboard. Build delegation-safe queries before complex rollups.
A
SharePoint Foundation
Create 6 lists, all columns, choice values, indexed columns, M365 groups. Create Evidence document library. Set list-level permissions by group.
B
Power Apps Shell
Create Canvas App. Connect all 6 lists as data sources. Build navigation shell with 5 screens. Implement role detection on OnStart.
C
Power Apps My Queue
Build delegable gallery with owner filter. Show: my findings, due soon, overdue, missing evidence. Prove the workflow works.
D
Power Apps Finding Detail
Build center of gravity: summary, review, patch, evidence, history tabs. Wire child record queries. Add closure button guard.
E
Power Apps Review Board + Admin
Build board queue, three-panel detail, decision controls. Build admin screen for reference data management.
F
Power Automate Core Flows
Build flows 1–4: owner assigned, review requested, closure approval, reopen alert. These drive the state machine.
G
Power Automate Support Flows
Build flows 5–7: weekly digest, KPI rollup, evidence lock. These harden governance.
H
Power BI Analytics
Build semantic model, DAX measures, 4 report pages. Only after workflow is stable. Embed Power Apps visual for action-from-analytics.

What to Postpone (Not in V1)

Full memory graph visualization Granular claim/event normalization Item-level security everywhere Complex cross-list rollup inside Power Apps PDF packet generation inside the app
Handle rollups with flows or a summary list. Move to Dataverse when SharePoint becomes the constraint.
