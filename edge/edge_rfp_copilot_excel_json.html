What This Is

A repeatable, AI-assisted workflow that turns any government RFP into structured data your entire team can use — without re-reading 200 pages.

RFP Docs → Co-Pilot AI → {} JSON → Excel Tables → Team Executes

The Problem

Every role on a proposal team re-reads the same RFP, highlights different things, and maintains their own notes. When an amendment drops, everyone scrambles independently. Drift happens. Things get missed.

The Solution

  1. Parse once — feed the RFP to Co-Pilot with a structured extraction prompt
  2. Get JSON — Co-Pilot returns a single, structured JSON file with citations
  3. Load into Excel — Power Query turns JSON into live tables: dates, requirements, attachments, risks
  4. Team pulls what they need — each role has a pre-defined action packet, no re-parsing
  5. Amendment arrives? Rerun. — overwrite JSON, Refresh All in Excel, team stays aligned
The Refresh Loop
Amendment arrives → Rerun prompt → Overwrite JSON → Excel Refresh All → Team stays aligned.
Bottom line: One person parses. The whole team benefits. Amendments don't break the workflow — they feed it.

George Quick Start — 3 Minutes to Structured Data

This is the fastest path. You need: the RFP PDF, Microsoft Co-Pilot (or any LLM), and Excel.

Step 1: Copy the Prompt (30 seconds)

  1. Go to the Co-Pilot Prompt tab
  2. Click Copy Prompt
  3. Open Co-Pilot (or ChatGPT, Claude, etc.)
  4. Paste the prompt, then attach or paste the RFP document

Step 2: Save the JSON (30 seconds)

  1. Co-Pilot returns a JSON block — copy it all
  2. Open Notepad (or any text editor)
  3. Paste and save as rfp_extract.json on your desktop or shared drive
  4. Verify: file should start with { "solicitation":

Step 3: Load into Excel (2 minutes)

  1. Open a new Excel workbook
  2. Go to Data → Get Data → From File → From JSON
  3. Select your rfp_extract.json
  4. Power Query opens — expand the solicitation record
  5. Click Close & Load to create the table
  6. Or paste the Power Query M script from the Excel Power Query tab for pre-built tables
Pro tip: Save the workbook. Next amendment? Overwrite the JSON file, then click Data → Refresh All. Tables update instantly.

Step 4: Share with the Team

  1. Drop the JSON file in a shared folder (Teams, SharePoint, network drive)
  2. Send a one-liner email: "RFP extracted — JSON is at [path]. Pull what you need."
  3. Each role pulls their action items from the Roles + Actions tab
Done. Three minutes. Whole team aligned.
Next time an amendment drops, repeat steps 1-2, then Refresh All in Excel.

What This Is

Pre-built Power Query (M) scripts you paste into Excel's Advanced Editor. Each script loads a specific section of the JSON into a clean table. One JSON file → multiple Excel tables → Refresh All when data changes.

Setup Instructions

  1. Save rfp_extract.json to a known path (e.g. C:\RFP\rfp_extract.json)
  2. In Excel: Data → Get Data → From Other Sources → Blank Query
  3. In the formula bar, click Advanced Editor
  4. Paste one of the M scripts below, update the file path, click Done
  5. Click Close & Load to create the table
  6. Repeat for each table you want
Refresh All: When you overwrite the JSON file (after an amendment rerun), just press Ctrl+Alt+F5 or Data → Refresh All. All tables update from the same file.

Pre-Wired XLSX Layout Spec

Key Datestbl_KeyDatessolicitation.key_dates[]
Attachmentstbl_Attachmentssolicitation.attachments_inventory[]
Amendmentstbl_Amendmentssolicitation.amendments[]
Riskstbl_Risksrisks[]
Open Itemstbl_OpenItemsopen_items[]
Binding the JSON path: Each query references the same file path. Change it once in each query's first line. For team use, point all queries to a shared network path (e.g. \\server\rfp\rfp_extract.json) so everyone refreshes from the same source.

Query 1: Solicitation Overview

let
    // UPDATE THIS PATH to your local or shared JSON file
    FilePath = "C:\RFP\rfp_extract.json",

    Source = Json.Document(File.Contents(FilePath)),
    sol = Source[solicitation],

    // Extract flat fields into a single-row record
    Record = [
        title                  = sol[title],
        solicitation_number    = sol[solicitation_number],
        agency                 = sol[agency],
        naics                  = sol[naics],
        set_aside              = sol[set_aside],
        contract_type          = sol[contract_type],
        period_of_performance  = sol[period_of_performance],
        place_of_performance   = sol[place_of_performance],
        eval_method            = sol[evaluation][method],
        submission_format      = sol[submission][format],
        delivery_method        = sol[submission][delivery_method]
    ],

    AsTable = Record.ToTable(Record),
    Transposed = Table.Transpose(AsTable),
    Headers = Table.PromoteHeaders(Transposed, [PromoteAllScalars=true])
in
    Headers

Query 2: Key Dates

let
    FilePath = "C:\RFP\rfp_extract.json",
    Source = Json.Document(File.Contents(FilePath)),
    dates = Source[solicitation][key_dates],
    AsTable = Table.FromList(dates, Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    Expanded = Table.ExpandRecordColumn(AsTable, "Column1",
        {"event", "date", "citation"},

    Expanded

What This Is

Pre-defined action packets for each proposal role. After the JSON is shared, each person pulls their section and executes — no full-RFP re-read required.

Proposal Manager

Focus: Timelines, volumes, page limits, submission logistics

Pull from JSON: key_dates, submission, evaluation.factors

  • Build proposal calendar from key_dates
  • Assign volume leads based on submission.volumes
  • Set page limits per volume
  • Schedule color team reviews (Pink/Red/Gold)
  • Configure submission method and format

Compliance Lead

Focus: Section L/M requirements, compliance matrix, mandatory forms

Pull from JSON: compliance_matrix, evaluation, attachments_inventory

  • Map compliance_matrix requirements to proposal outline
  • Flag mandatory certifications and representations
  • Build Section L response checklist
  • Cross-reference eval factors with response sections
  • Track attachment completion status

Technical Lead

Focus: SOW/PWS scope, deliverables, technical evaluation factors

Pull from JSON: scope, evaluation.factors, compliance_matrix (technical reqs)

  • Draft technical approach aligned to scope.key_requirements
  • Map approach to evaluation factors (highest-weight first)
  • Identify deliverables and milestone schedule
  • Determine staffing and SME needs
  • Flag technical risks from risks array

Cost / Pricing Lead

Focus: Pricing structure, labor categories, cost evaluation method

Pull from JSON: evaluation.factors (cost weight), scope, attachments_inventory (price template)

  • Build cost model from labor category matrix (Attachment B)
  • Map LOE estimates to task order structure
  • Determine fee/profit structure per contract type
  • Draft basis of estimate (BOE) narrative
  • Cross-check cost eval method (LPTA vs. trade-off)

Contracts Lead

Focus: Contract type, terms and conditions, FAR/DFARS clauses

Pull from JSON: solicitation.contract_type, solicitation.set_aside, open_items

  • Review contract type implications (IDIQ, FFP, T&M)
  • Flag non-standard FAR/DFARS clauses
  • Prepare reps, certs, and required forms
  • Review set-aside and subcontracting requirements
  • Resolve open_items with Contracting Officer

Staffing / HR Lead

Focus: Labor categories, clearance requirements, key personnel, PoP locations

Pull from JSON: scope, attachments_inventory (labor cat matrix), risks (staffing risks)

  • Map labor categories to available resumes
  • Verify clearance levels (Secret, TS/SCI, etc.)
  • Identify key personnel requirements and candidates
  • Flag staffing gaps — initiate recruiting if needed
  • Confirm place of performance and remote/onsite mix
Team Pull Model: Nobody re-reads the full RFP. Each role pulls their JSON keys, gets their action items, and executes. The proposal manager orchestrates; the data source stays consistent.

What This Is

Common problems you'll hit and how to fix them. Most issues are JSON formatting or PDF quality problems.

JSON Malformed

ERROR Excel or a JSON validator says the file is invalid.

CauseFix
Co-Pilot wrapped output in markdown fences (```json ... ```)Remove the ```json and ``` lines. File must start with { and end with }
Trailing comma after last array/object itemDelete the trailing comma. Use jsonlint.com to find the exact line
Smart quotes ("") instead of straight quotes ("")Paste into a plain text editor first (Notepad), then save. Avoid Word/Outlook paste.

Missing 'solicitation' Root Key

WARN Power Query can't find the solicitation field.

CauseFix
Co-Pilot returned a flat structure without nestingRe-run the prompt. Emphasize: "Return a single JSON object with solicitation as the top-level key."
Co-Pilot returned multiple JSON blocksKeep only the first complete { ... } block. Discard commentary.

Excel Can't Parse Nested Arrays

WARN Power Query shows "List" or "Record" instead of values.

CauseFix
You loaded the root JSON without expandingIn Power Query, click the expand icon (double arrow) on columns showing "List" or "Record"
Deeply nested arrays (e.g. arrays inside arrays)Use the specific M scripts from the Excel Power Query tab — they handle expansion automatically

Amendments Conflicting

WARN Amendment changes a date or requirement that contradicts the base solicitation.

CauseFix
Amendment supersedes original dates/termsRe-run the prompt with the base solicitation + all amendments attached. The prompt asks Co-Pilot to incorporate amendments into the base data.
Multiple amendments with conflicting infoFeed amendments in order (0001, 0002, ...). Tell Co-Pilot: "Latest amendment takes precedence."

Scanned PDF — Not Machine Readable

ERROR Co-Pilot returns empty or garbled output.

CauseFix
PDF is a scanned image, not searchable textRun OCR first: Adobe Acrobat → Scan & OCR → Recognize Text. Then re-upload.
PDF has copy protectionPrint to PDF (removes restrictions), then upload the new copy.
Tables are images, not text tablesExtract tables manually or use a table-extraction tool (Tabula, Camelot) before feeding to Co-Pilot.

Security: Protect Proprietary Information

CRITICAL Do not paste internal notes, pricing data, competitive intelligence, or proprietary strategies into any external AI tool.

Safe to paste: The government-issued RFP document (it's public after release).
Do NOT paste: Internal pricing models, win themes, competitive analysis, teammate proprietary data, personnel PII, draft proposal content, or anything marked CUI/FOUO.
If using Co-Pilot in Microsoft 365: Data stays within your tenant boundary. Check with your IT/security team for policy confirmation.
